## 🐛 Common Issues and Fixes

### ❌ GUI doesn't appear after execution
**Fix:** Confirm you are inside an active Grow a Garden session, not the main menu or loading screen. Detach your executor, reattach, and re-execute. If using a cached script, clear the executor's script history and paste fresh.

### ❌ Auto Farm plants seeds but never harvests
**Fix:** Open the Farm tab and confirm **Auto Harvest** is toggled ON separately from **Auto Plant** — they are independent toggles. Also check that your farm plot is within the auto-detect radius (default: 150 studs from your character).

### ❌ Pet Spawner shows "Spawn Failed" for certain pets
**Fix:** Some event pets are only spawnable during their active event window. If the event has ended, those pets are locked. For non-event pets showing this error, update to the latest script version — pet ID tables are updated with each Grow a Garden content patch.

### ❌ Dupe Pet / Egg Duper produces no output
**Fix:** The dupe modules require at least one copy of the target item already in your inventory. You cannot dupe from zero. Also ensure you are in a private server — public servers with high player counts sometimes have server-side dupe protections active.

### ❌ Script crashes immediately on Android
**Fix:** Ensure you're running the latest `grow a garden script android` build and that your executor version is 3.0 or higher. Fully uninstall the old executor version before reinstalling — cached data from older versions causes injection conflicts.

### ❌ Admin Panel weather override reverts after 30 seconds
**Fix:** Enable the **Lock Weather** toggle inside the Admin Panel after setting your preferred weather type. Without it, the game's weather cycle will override the setting on its next tick.

### ❌ Mutation Whitelist is harvesting non-mutated crops
**Fix:** Open the Farm tab, scroll to Mutation Whitelist, and confirm the whitelist is set to **Mutated Only** and not **All Crops**. After a script update, this setting resets to default — re-enable it each session.
